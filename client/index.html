Inertial Positioning

Based on inertial sensors and GPS locations

About the project

Using the GPS with mobile phones is very battery consuming. And that’s the reason why users don’t want to use GPS. However, GPS is still the best and easiest way to locate the location of the user. In this project we try to develop model which leverage inertial sensors of mobile phones to keep or even increase the location accuracy when reducing the GPS frequency. Usually location data also contains inaccurate location points and thus, another goal is to find way to detect these possible outliers.

  • Reduce the polling frequency of GPS coordinates
  • Detect outlying positions
  • Detect position without GPS available

Preprocessing

There are three main challenges when dealing with sensor data in this context: The frequency of the different sensors is usually different and contains a lot of missing data; sensors accumulate various type of noises, which can be difficult to deal with; and finally, data from the sensors is collected relative to the position of the phone.

To deal with the different frequencies, we explore the distribution of frequencies for the accelerometer data, select the value that best suits the observations, and finally re-sample the different sensors, dealing with missing data by interpolating the values. Then, the denoising section of the preprocessing is carried out by applying a filter to the synchronized data. The relative position can be generalized by rotating the entries using the gyroscope data as a baseline.

Prediction

In this project, we’re using Long-Short-Term-Memory networks to predict a route based on sensor data collected by mobile devices. LSTM is a type of Recurrent Neural Networks (RNN), which are able to handle sequential data. Location and sensor data are sequential time-series data in which the next point is dependent from the previous one.

Data modelling task is a regression type. A next point is predicted by sensor data during a timestep. Location points could be taken from GPS but it uses battery life a lot. Also, there are significant inaccuracies in GPS especially indoors spaces such as shopping centers or metro. That is why experimenting possibilities of using sensor data instead of GPS would benefit positioning in many ways.

Two models are used. A one for predicting difference in latitude and another for longitude. The models used in the plot below takes as an input a single row of predicting variables. At this point only preprocessed accelerometer data was used. The hidden layer consist of a single LSTM block with a default sigmoid activation function. The output layer produces a single continuous output value, which represents the difference between a previous point and a next point.

It can be seen from the plot below that the model is able to find the direction of the real path with a really little training. There are naturally significant inaccuracies since the model is quite simple and has been trained with a small amount of data so far.
